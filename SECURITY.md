# Security Policy

## Supported Versions

Security updates are provided for the latest released minor version on the
default branch. Older versions are supported on a best-effort basis.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < latest| :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, report them privately using GitHub's
[private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)
feature (the **Security** tab of this repository), or by emailing the
maintainer.

Please include:

- A description of the vulnerability and its impact
- Steps to reproduce or proof of concept
- Affected versions and configuration
- Any suggested remediation

## Response Targets

| Stage                | Target        |
| -------------------- | ------------- |
| Acknowledgement      | 3 business days |
| Triage & severity    | 7 business days |
| Fix or mitigation    | Based on severity (CVSS) |

We follow a coordinated disclosure process and will credit reporters unless
anonymity is requested.
